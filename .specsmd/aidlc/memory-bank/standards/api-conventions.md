# API Conventions

## No Custom REST API

There is no hand-rolled API server. Angular services call the **Supabase JS client** directly. Edge Functions handle server-side orchestration only (notification triggers, IMC import, eDocs upload URL generation).

## Supabase Client Setup

```typescript
// lib/supabase.ts — single client instance, shared via injection token
import { createClient } from '@supabase/supabase-js';
import { Database } from './database.types'; // generated by `supabase gen types typescript`

export const supabase = createClient<Database>(
  environment.supabaseUrl,
  environment.supabaseAnonKey
);
```

Never instantiate `createClient` in a component. Inject the singleton.

## Typed Queries

All queries use the generated `Database` type. Never write untyped `.from('table_name')` calls.

```typescript
// Good
const { data, error } = await supabase
  .from('cases')
  .select('id, title, status')
  .eq('hfa_id', hfaId);

// Bad — no type safety
const { data } = await supabase.from('cases').select('*');
```

Map the raw Supabase row type to a typed domain interface before passing to components:

```typescript
// Domain interface (not the raw Supabase row)
export interface Case {
  id: string;
  title: string;
  status: 'active' | 'closed';
}
```

## Service Pattern

Smart services own all Supabase calls. Components never call `supabase` directly.

```typescript
@Injectable({ providedIn: 'root' })
export class CaseService {
  private supabase = inject(SUPABASE_CLIENT);

  async getCases(hfaId: string): Promise<Case[]> {
    const { data, error } = await this.supabase
      .from('cases')
      .select('...')
      .eq('hfa_id', hfaId)
      .throwOnError();
    return data.map(toCaseDto);
  }
}
```

Use `.throwOnError()` on queries so errors propagate as exceptions rather than silent nulls.

## Realtime Subscriptions

Centralise all Supabase Realtime subscriptions in `RealtimeService`. Components never call `.channel()` directly.

```typescript
@Injectable({ providedIn: 'root' })
export class RealtimeService {
  subscribeToCase(caseId: string, handler: (event: RealtimeEvent) => void): RealtimeChannel {
    return this.supabase
      .channel(`case:${caseId}`)
      .on('postgres_changes', { event: '*', schema: 'public', table: 'conversation_messages',
          filter: `case_id=eq.${caseId}` }, handler)
      .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'prerequisites',
          filter: `case_id=eq.${caseId}` }, handler)
      .subscribe();
  }
}
```

Subscribe in `ionViewDidEnter` or `ngOnInit`. Unsubscribe in `ionViewWillLeave` or `ngOnDestroy` by calling `channel.unsubscribe()`.

## Auth

```typescript
// Sign in (passwordless)
await supabase.auth.signInWithOtp({ email });

// Verify OTP
await supabase.auth.verifyOtp({ email, token, type: 'email' });

// Get current user
const { data: { user } } = await supabase.auth.getUser();

// Sign out
await supabase.auth.signOut();
```

Store the session in `@capacitor/preferences` (not localStorage) before any production build. For the hackathon, Supabase's default session persistence is acceptable.

## Edge Functions

Call Edge Functions for operations that require service-role access or orchestration across multiple systems (IMC import, eDocs upload URL generation, notification dispatch):

```typescript
const { data, error } = await supabase.functions.invoke('import-imc-project', {
  body: { imcProjectId: '...' }
});
```

Edge Functions receive the user's JWT automatically — no need to pass it manually.

## Error Handling

- Use `.throwOnError()` on all queries so errors bubble to the service layer
- Services catch Supabase `PostgrestError` and map to typed application errors
- Never swallow errors silently; surface them to the user or log them

## Environment Variables

```typescript
// environment.ts (not committed — .gitignore)
export const environment = {
  supabaseUrl: 'https://xyz.supabase.co',
  supabaseAnonKey: 'eyJ...',
};
```

Never hardcode keys in service files. Never commit `environment.prod.ts` with real keys.
